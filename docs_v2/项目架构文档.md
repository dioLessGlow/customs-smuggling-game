# 缉私十二时辰 · 项目架构文档 v3.0

> 反走私创意大赛教育游戏 — 36关 × 6导师 × 周期系统 × 商店系统
> 去生肖化设计 | 纯静态H5 | Phaser 3 + amCharts 5 + Canvas 2D

---

## 一、项目概览

| 项目 | 值 |
|------|-----|
| **游戏名称** | 缉私十二时辰 |
| **核心定位** | 反走私宣传第一，游戏性第二 |
| **参赛目标** | 第四届「穗澳杯」珠海·澳门大学生反走私创作大赛 |
| **屏幕基准** | 390 × 844 (iPhone 14 竖屏) |
| **缩放策略** | 移动端 `window.innerWidth/390` 全屏适配；桌面端 `Math.min(1, w/390, h/844)` 居中模拟手机 |
| **渲染层** | Phaser 3 (游戏场景) + amCharts 5 (地图) + Canvas 2D (潮汐背景) |
| **技术栈** | ES6+ 原生 JavaScript，零构建步骤，纯 CDN 加载 |
| **存储方案** | localStorage (key: `anti_smuggling_save_v4`) + `_memoryStore` 内存兜底 |
| **外部依赖** | amCharts 5 (地图)、Phaser 3.70 (游戏引擎) |
| **部署方式** | GitHub Pages / Gitee Pages / Vercel，纯静态单页 |

```javascript
const defaultSave = {

  // === 周期系统 ===

  // === 直觉系统（商店接管） ===
  echoSkills: [],                                      // 已获取直觉ID列表
  currentEcho: null,                                   // 当前装备直觉ID
  chosenCharacterId: null,                             // 已选角色ID（保留字段）

  // === 收集系统 ===
  fragments: [],                                       // 已收集剧情碎片ID
  archive: [],                                         // 图鉴条目ID列表
  seenItems: [],                                       // 已见物品记录

  // === 商店系统 v3.2 ===
  shopItems: [],                                       // 已购装备/技能/直觉ID
  shopKnowledge: [],                                   // 已购知识点ID
  quizResults: {},                                     // { k01: 5 } 测验奖励记录
  firstWinBonuses: [],                                 // 首胜奖励领取记录
  hasShared: false,                                    // 是否已分享

  // === 每日登录 ===
  lastLoginDate: '',                                   // 上次登录日期 YYYY-MM-DD
  loginStreak: 0,                                      // 连续登录天数

  // === 设置 ===
  settings: {
    sound: true,                                       // 音效开关
    vibration: true                                    // 震动开关（移动端）
  },

  lastPlayed: Date.now()                               // 上次游玩时间戳
};
```

### 3.2 存档迁移策略

SaveManager._migrate() 处理历史版本兼容：

| 旧格式特征 | 处理方式 | 保留数据 |
|-----------|---------|---------|
| 含下划线ID（如 `human_rat`） | 重置为默认，保留功勋/碎片/设置 | 永久功勋、已收集碎片、设置 |
| 无下划线但缺商店字段 | 自动补齐 `shopItems`/`shopKnowledge`/`quizResults` | 全部 |
| 完整v4格式 | 直接加载 | 全部 |

## 七、关卡流程详解（GenericLevelScene）

### 7.1 生命周期

```
init(data)
  ├─ _findConfig()              ← 从LEVELS按levelId查找配置
  ├─ _resetState()              ← 重置所有游戏状态变量
  │   └─ _applyShopEffects()    ← 应用已购装备/技能/知识效果
  ├─ _buildItemPool()           ← 按场景类型选择物品池（12生肖池）
  └─ save = SaveManager.load()  ← 读取存档

create()
  ├─ _showDayIntro()            ← "第N天"动画（1.5s）
  ├─ _createHUD()               ← 创建HUD：标题/时间/进度/连击/分数
  ├─ _createScanArea()          ← 创建扫描区：行李/扫描框/扫描线/检测报告/深度检查按钮
  ├─ _createButtons()           ← 创建三按钮：放行/检查/扣押
  ├─ _createEchoIndicator()     ← 创建直觉状态指示器
  └─ _showItem()                ← 延迟1.5s后开始第一件物品

游戏循环（每物品）：
_showItem()
  ├─ 行李入场动画（0.3s缩放）
  ├─ 扫描线划过（1s）
  ├─ _showDetectionReport()     ← 显示安检仪结果（绿色/红色/橙色）
  └─ 等待玩家判定

玩家判定：
_judge(choice)
  ├─ choice === 'pass'（放行）
  │   ├─ 合法物品 → ✅正确：+10分，+1功勋，combo++，绿色粒子
  │   └─ 走私物品 → ❌错误：-5分（毒品-25），combo重置，红色闪烁
  │
  ├─ choice === 'inspect'（深度检查）
  │   └─ _secondaryCheck()      ← 2秒详细检查，-3秒时间，显示更多信息
  │
  └─ choice === 'seize'（扣押）
      ├─ 走私物品 → ✅正确：+10分，+1功勋，combo++，印章动画
      └─ 合法物品 → ❌错误：-5分，combo重置，红色闪烁

判定后直觉钩子：
  ├─ crisis_exempt: 错误时20%免罚
  ├─ combo_keep: 连击中断时30%保留
  ├─ adversity_rebound: 连续错2后下次正确+5
  └─ comboProtected: 3连击后触发"缉私直觉"，下次错误免罚

_onTick()（每秒执行）
  ├─ 时间-1
  ├─ wind_warning: 最后5秒高亮走私品
  ├─ time_mastery: 超时30%概率暂停5秒
  └─ 时间≤0 → _endLevel()

_endLevel()
  ├─ 计算最终得分
  ├─ lucky_bonus: +10%周期功勋
  ├─ 判定通过/失败（correctCount ≥ required）
  ├─ 通过 → 解锁下一关（同导师内线性解锁）
  ├─ CycleManager.addRound()    ← 记录本周期成绩
  └─ 派发 game:levelEnd 事件 → UI层接收显示result屏
```

### 7.2 物品判定规则

|----------|------------------|------------------|

## 八、周期系统（CycleManager）

### 8.1 核心规则

| 参数 | 值 |
|------|-----|
| 周期长度 | 10关（CYCLE_SIZE = 10） |
| 存活条件 | 周期内通关 ≥ 6关（60%通过率） |
| 通过奖励 | 周期号+1，随机解锁1个新直觉，周期功勋清零 |
| 失败惩罚 | 周期内功勋清零（`lostDao`），保留永久功勋和已拥有直觉 |

### 8.2 状态流转

```
周期开始（currentCycle = N, roundsInCycle = []）
  │
  ├─ 玩家选择关卡 → 游玩 → 通过/失败
  │   └─ addRound(levelId, passed, score, dao)
  │       ├─ push到roundsInCycle
  │       └─ 检查是否达到10关
  │
  ├─ 未满10关 → 继续选择下一关
  │
  └─ 满10关 → endCycle()
      ├─ 统计passedCount
      ├─ passedCount ≥ 6
      │   ├─ success = true
      │   ├─ currentCycle++
      │   ├─ roundsInCycle = []
      │   ├─ cycleDao = 0
      │   ├─ _unlockRandomEcho() → 随机解锁1个直觉
      │   └─ 返回 { success:true, cycle:N, passed:X, total:10, newEcho:ID }
      │
      └─ passedCount < 6
          ├─ success = false
          ├─ roundsInCycle = []
          ├─ cycleDao = 0（清零）
          └─ 返回 { success:false, cycle:N, passed:X, total:10, lostDao:Y }
```

### 8.3 周期结算UI

- **成功**：金色主题，"周期突破！"，显示新解锁直觉，周期号+1
- **失败**：暗红主题，"周期重置"，显示失去的功勋，保留永久进度提示

## 十一、音效系统（SoundManager）

### 11.1 技术方案

纯 Web Audio API 合成，零音频文件，无外部资源依赖。

### 11.2 音效清单

| 音效名 | 波形 | 频率 | 时长 | 触发时机 |
|--------|------|------|------|---------|

| **wrong** | sawtooth | 150→100Hz | 0.5s | 判断错误 |

| **echo** | sine | 660→1320Hz | 0.6s | 直觉触发 |

## 十二、每日登录系统（SaveManager.checkDailyLogin）

### 12.1 规则

- 每次 `init.js` 启动时自动调用
- 比较 `lastLoginDate` 与今日日期（YYYY-MM-DD）
- **连续登录**：`loginStreak++`
- **中断登录**：`loginStreak = 1`
- **每7天奖励**：`permanentDao += 10`

### 12.2 流程

```
init.js 启动
  └─ SaveManager.init()
      └─ checkDailyLogin()
          ├─ 读取 lastLoginDate
          ├─ 与今日日期比较
          ├─ 同日 → 无操作
          ├─ 次日 → loginStreak++, 检查是否满7天
          │   └─ 满7天 → permanentDao += 10, loginStreak = 0
          └─ 隔日以上 → loginStreak = 1
          └─ 更新 lastLoginDate = 今日
```

## 十五、关键约定与规范

### 15.1 命名规则

| 类型 | 规范 | 示例 |
|------|------|------|
| 关卡ID | `k` + 两位数字 | `k01`, `k36` |
| 导师ID | 全小写拼音 | `linrui`, `chenfeng`, `xiaohui` |
| 商品ID | 下划线命名 | `magnifier_1`, `crisis_exempt` |
| 效果类型 | 驼峰命名 | `tutorAlarm`, `tutorTime`, `globalAlarm` |
| 存档键 | 常量定义 | `anti_smuggling_save_v4` |

### 15.2 坐标与DPR

| 场景 | 处理方式 |
|------|---------|
| Phaser游戏 | `resolution: window.devicePixelRatio` |
| 潮汐Canvas | `canvas.width = 390 * dpr; ctx.setTransform(dpr, 0, 0, dpr, 0, 0)` |
| DOM UI | CSS `transform: scale()` 由 `fitApp()` 统一计算 |
| 逻辑坐标 | 全部基于 390×844，不硬编码DPR |

### 15.3 去生肖化约定

| 层面 | 处理方式 |
|------|---------|
| 用户可见文字 | 完全无"生肖"概念，使用"类型""领域"等替代 |
| 代码内部 | 保留 `zodiac` 字段仅为物品池选择逻辑，不暴露给用户 |
| 关卡命名 | 使用2字主题名（审名、搜隐、截浪等），无生肖关联 |
| 导师命名 | 直接使用姓名，无生肖映射 |

### 15.4 加载顺序铁律

```
config/（数据层，无依赖）
  → renderers/（潮汐，独立运行）
  → managers/（逻辑层，依赖config）
  → scenes/（Phaser场景，依赖managers+config）
  → ui/（DOM界面，依赖所有以上，init.js最后执行）
```

### 15.5 数据源唯一性

| 数据 | 唯一数据源 | 消费方 |
