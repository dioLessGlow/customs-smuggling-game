# 缉私终焉 · 全面开发规划文档

> **项目代号**：缉私终焉  
> **版本**：v2.1 规划版  
> **技术栈**：Phaser 3 + 原生 JavaScript + GitHub Pages  
> **目标平台**：手机竖屏（主）/ 电脑浏览器（兼容）  
> **开发周期**：个人参赛，3-4周迭代  
> **参赛**：第四届"思创杯"珠澳大学生反走私创意大赛

---

## 一、项目概述

### 1.1 核心定位

《缉私终焉》是一款以珠澳口岸反走私为主题的**竖屏H5策略识别游戏**。游戏深度融合番茄小说《十日终焉》的"规则扭曲+轮回重置"机制，将反走私知识科普与游戏化体验结合，打造"玩游戏学反走私"的创新参赛作品。

### 1.2 差异化卖点

| 竞品/常规方案 | 缉私终焉 |
|-------------|---------|
| 干巴巴的答题游戏 | **规则扭曲** — 表面规则下隐藏反直觉设定 |
| 一次性通关即弃 | **十日轮回** — 失败重置但保留记忆（解锁内容） |
| 无情感连接 | **剧情碎片** — 拼凑反走私大案，追更驱动 |
| 单一机制 | **回响觉醒** — 极端情绪下触发随机技能 |
| 无成长感 | **道系统** — 局内积累+局外永久成长 |

### 1.3 目标用户

- **核心**：珠澳大学生（参赛评审直接受众）
- **扩展**：18-30岁手游玩家，偏好策略/解谜类
- **传播**：竖屏H5，微信扫码即玩，适合社交分享

## 四、核心机制详解

### 4.1 基础识别机制

**流程：**
1. 行李入场（0.3秒缩放动画）
2. 扫描线划过（1秒）
3. 显示物品名称
4. 玩家判断：放行/扣押
5. 即时反馈（粒子+数字+音效）

**反馈矩阵：**

| 操作 | 物品类型 | 结果 | 得分 | 特效 |
|-----|---------|------|------|------|
| 放行 | 合法 | ✅正确 | +10 | 绿色粒子+清脆音效 |
| 扣押 | 非法 | ✅正确 | +10 | 绿色粒子+印章动画 |
| 放行 | 非法 | ❌错误 | -5 | 红色闪烁+屏幕震动 |
| 扣押 | 合法 | ❌错误 | -5 | 红色闪烁+屏幕震动 |

### 4.2 隐藏规则系统

**触发条件：** 第3件物品正确扣押时

**揭示方式：** 弹出"💡 隐藏规则发现"卡片

**人鼠关示例：**
> "走私者会故意在前2件放合法物品，消耗你的注意力。第3件往往是真的！"

**设计目的：**
- 降低首次失败挫败感（"原来如此"的顿悟）
- 增加重玩价值（知道规则后策略改变）
- 契合反走私主题（走私者的心理战术）

### 4.3 十日轮回系统

**规则：**
- 每周期 = 10关（不限于人级，可跨级别）
- 通关条件：周期内通关 ≥ 6关
- 成功：进入下一周期，随机觉醒1个回响
- 失败：周期内积累的"道"清零，保留永久道

**保留内容：**
- ✅ 已觉醒的回响技能
- ✅ 已收集的剧情碎片
- ✅ 已解锁的图鉴
- ✅ 永久道
- ❌ 周期内的道
- ❌ 当前周期进度

### 4.4 回响系统（6种）

| 回响 | 图标 | 效果 | 触发条件 |
|-----|------|------|---------|
| **灵闻** | 👂 | 提前显示物品类别（可疑/正常） | 被动，始终生效 |
| **替罪** | 🛡️ | 本次错误不扣分 | 错误时自动触发 |
| **招灾** | 🌪️ | 扣分减半 | 错误时自动触发 |
| **天行健** | ⏸️ | 时间暂停5秒 | 主动/被动触发 |
| **生生不息** | ♾️ | 答错不中断连击 | 错误时自动触发 |
| **强运** | 🍀 | 额外一次重试 | 关卡失败时触发 |

**觉醒方式：** 周期通关后随机解锁1个

### 4.5 放大镜机制（虎牙关）

**规则：**
- 每关3次使用机会
- 使用后显示物品真实类别（⚠️可疑 / ✅正常）
- 不使用时所有物品显示"? ? ?"

**策略深度：**
- 何时使用？（早期消耗 vs 留到后期）
- 对可疑物品使用 vs 对不确定物品使用

### 4.6 风险差异化（蛇毒关）

| 错误类型 | 扣分 | 特效 |
|---------|------|------|
| 普通走私品错误 | -5 | 红色闪烁+震动 |
| **毒品走私错误** | **-50** | **屏幕剧烈震动+警报声+红色全屏** |

**缉私直觉：** 连续正确3次后，下次错误不扣分（金色光环提示）

## 七、音效设计

### 7.1 程序生成音效（Web Audio API）

| 音效 | 波形 | 频率变化 | 时长 | 触发时机 |
|-----|------|---------|------|---------|
| correct | sine | 880→1760Hz | 300ms | 判断正确 |
| wrong | sawtooth | 150→100Hz | 500ms | 判断错误 |
| alarm | square | 440↔880Hz | 600ms | 毒品错误 |
| click | sine | 600→800Hz | 100ms | 按钮点击 |
| levelup | sine | 523→1047Hz | 400ms | 关卡通过 |
| scan | square | 1200→1800Hz | 200ms | 扫描开始 |
| echo | sine | 660→1320Hz | 600ms | 回响觉醒 |
| cycle | sine | 523→659→784→1047Hz | 600ms | 周期通关 |

### 7.2 音频策略

- **无背景音乐**：突出音效反馈，减少包体
- **首次交互解锁**：浏览器策略要求用户点击后初始化 AudioContext
- **降级处理**：音频失败不阻断游戏流程

## 九、剧情碎片设计

### 9.1 叙事结构

12个碎片拼成完整反走私大案：

```
碎片1(鼠): 发现水客集团线索
    ↓
碎片2(牛): 冻品走私链条
    ↓
碎片3(虎): 象牙雕刻上的符号 → 跨国网络
    ↓
碎片4(兔): 异宠体内藏蛇卵
    ↓
碎片5(龙): 枪支配件+纸条"澳门十六铺"
    ↓
碎片6(蛇): 体内藏毒者说出"天龙"
    ↓
碎片7(马): 伪报单→神秘账户
    ↓
```

```bash
# 1. 创建仓库
git init
git remote add origin https://github.com/yourname/anti-smuggling-game.git

# 2. 提交代码
git add index.html game.js README.md

# 3. 启用 Pages
```
